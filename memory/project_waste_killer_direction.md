# Refined Product Direction — Waste-Killer for Bakery Chains

## Type

project

## Direction

ObradorIQ was sharpened from a vague "AI for bakery production" into a specific wedge:

- **Product:** waste-reduction tool for **small bakery chains (2–4 sites)** that increases
  profit directly. Frame everything as **euros of waste avoided = profit recovered**.
- **Differentiator (the moat):** cross-site **plan-level reallocation** — reallocate
  *planned production quantities* across sites (NOT physical goods, advisory only). Impossible
  for a solo bakery; not modeled by generic forecasting SaaS.
- **First user:** owner-operator of a 2–4 site chain. **Effort goal:** an excellent, deployed
  class submission, not a real launch.
- **Daily hook = waste; weekly escalation = profit** (waste-adjusted "True" margin).

## Why

The original idea was a vitamin doing everything at once, with a fatal data-entry cold-start
problem. Narrowing to waste (the painkiller) + the multi-site reallocation edge makes it
defensible and demoable. Decisions locked: reallocation is plan-level + advisory; build is
contract-first parallel tracks. See `[[project_agent_framework]]` and `[[project_tech_stack]]`.

## How to apply

Canonical docs: `docs/ideas/obradoriq-waste-killer.md` (one-pager with Not-Doing list),
`AGENT_FRAMEWORK.md`, `TECH_SPEC.md`. The "Not Doing" list is binding — resist scope creep
back toward a platform (no solo-baker market, no ingredient inventory, no sell-more,
no physical transfer logistics, no real-time POS in v1).
