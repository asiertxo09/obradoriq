# Class Submission Requirements

## Type

project

## Requirement

The project submission should include everything known so far about ObradorIQ:

- Problem statement: one short paragraph with a specific user and pain.
- Target user: who they are and the context where they experience the problem.
- Value proposition: the change the product creates for that user.
- First PRD covering product purpose, target user, user stories, core features, AI functionality, data inputs, success metric, and MVP scope.
- Initial architecture idea covering frontend, backend, database, AI, and automation.
- Generated Markdown files: `agents.md`, `memory.md`, and `todo.md`.

## Current project

ObradorIQ is an AI-powered optimization platform for bakeries and pastry shops. It helps bakery owners use sales, inventory, waste, and demand signals to make better daily production decisions.
